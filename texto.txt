
Q01. Coloque as quatro etapas do ciclo de vida de um programa C++ na ordem correta de ocorrência:
( 4) Execução
( 2) Compilação
( 1) Pré-processamento
( 3) Link

Resumo rápido:
•Pré-processamento: trata #include, #define etc.
•Compilação: transforma o código em linguagem de máquina (objeto).
•Link: junta arquivos e bibliotecas, criando o executável.
•Execução: o programa roda.

Q02. Um colega afirma: "O compilador lê o meu arquivo .cpp exatamente como eu escrevi, sem nenhuma modificação
prévia." Essa afirmação está correta? Justifique sua resposta mencionando a etapa responsável por processar diretivas
como #include 

Resumo + explicação:
Antes do compilador ler o código, existe a etapa de pré-processamento. É ela que faz alterações no arquivo .cpp.
	•	O pré-processador processa diretivas como #include, #define, #ifdef
	•	Por exemplo, o #include insere o conteúdo de outros arquivos (como bibliotecas) dentro do seu código
	•	Só depois disso o código “modificado” é enviado para a compilação

Q03. Qual é a diferença entre um arquivo objeto (.o ou .obj) gerado pelo compilador e o executável final (.exe ou
binário sem extensão) gerado pelo linker? Por que a etapa de linkagem é necessária?

Diferença:
	•	Arquivo objeto (.o / .obj):
É o código já compilado, mas incompleto. Pode ter funções/variáveis que ainda não foram “ligadas” (referências externas).
	•	Executável (.exe / binário):
É o arquivo final e completo, pronto para rodar, com tudo já conectado (código + bibliotecas).

Por que a linkagem é necessária?
A etapa de link (linkagem) junta todos os arquivos objeto e bibliotecas, resolvendo as referências (ex: funções de outras partes do programa ou da biblioteca padrão). Sem isso, o programa não funcionaria.

Resumo curto:
Objeto = código parcial
Executável = código completo pronto pra executar
Linkagem = conecta tudo para funcionar corretamente
 

Q04. Analise o trecho abaixo e diga em qual etapa do ciclo de vida o erro seria detectado e por quê:
#include <iostream>
int main() {
std::cout << "Olá" // falta o ponto e vírgula
return 0;
}

O erro é detectado na etapa de compilação.

Por quê?
Falta um ; no final da linha:

std::cout << "Olá"

Isso é um erro de sintaxe, e quem verifica a sintaxe do código é o compilador. Ele não consegue interpretar a instrução sem o ponto e vírgula e gera erro.

Resumo curto:
	•	Tipo de erro: sintaxe
	•	Etapa: compilação
	•	Motivo: código está escrito de forma incorreta para o compilador entender


Q06. O linker recebe a chamada std::cout no seu código. Onde ele vai buscar a implementação real dessa
funcionalidade para produzir o executável final?

O linker vai buscar a implementação do std::cout na biblioteca padrão do C++, geralmente a iostream (que faz parte da biblioteca padrão já compilada).

Explicação:
	•	Quando você usa #include <iostream>, o pré-processador só declara o que existe
	•	A implementação real (o código que faz o cout funcionar) está em bibliotecas já prontas do C++
	•	O linker conecta seu código com essas bibliotecas para formar o executável


Q07. Verdadeiro ou Falso — justifique cada item:
a. O pré-processador é parte do compilador C++ e entende a sintaxe da linguagem.
b. Um programa pode ter erros de linkagem mesmo que a compilação tenha sido bem-sucedida.
c. A etapa de execução transforma código-fonte em código de máquina.                                                                                                                                                                                           a) Falso

O pré-processador (como o #include, #define) atua antes da compilação.
Ele não entende a sintaxe completa do C++, apenas faz substituições de texto e inclusões de arquivos.
Ou seja, ele não “compreende” o programa como o compilador faz.

 Resumo: Pré-processador ≠ compilador, e não analisa sintaxe.

b) Verdadeiro

Um programa pode compilar sem erros, mas ainda falhar na linkagem.
Isso acontece quando, por exemplo, você declarou uma função mas não forneceu sua implementação, ou faltou alguma biblioteca.

Resumo: Compilou ok ≠ programa completo; pode faltar algo na hora de juntar tudo.

c) Falso

A transformação de código-fonte em código de máquina acontece na compilação, não na execução.
A execução é quando o programa já pronto é rodado pelo computador.

 Resumo:

Compilação → gera código de máquina
Execução → roda o programa


Q08. Um aluno criou dois arquivos: matematica.cpp (com funções de soma e subtração) e main.cpp (que chama
essas funções). Ele compilou apenas o main.cpp e recebeu um erro de linkagem. Explique o motivo do erro e como
corrigi-lo.                                                                                                                                                                                                                                                                                                           O erro aconteceu porque o aluno compilou apenas o main.cpp, mas as funções (soma e subtração) estão em matematica.cpp.

Durante a compilação, o código do main.cpp pode estar correto (as funções foram declaradas), porém na linkagem o compilador tenta juntar todas as partes do programa. Como o arquivo matematica.cpp não foi incluído, o linker não encontra as implementações das funções → erro.

Como corrigir:
Compilar os dois arquivos juntos:

g++ main.cpp matematica.cpp -o programa

Resumo:
Erro de linkagem ocorre porque faltou incluir o arquivo onde as funções estão implementadas.

Assim, ele interpreta a próxima linha como continuação da anterior, gerando erros em cascata.

Exemplo:

    int x = 10   // faltou ;

    return 0;
}

 Resumo:
Um único ; faltando “bagunça” a interpretação do código, gerando múltiplos erros aparentemente sem relação.
